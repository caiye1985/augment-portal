# 套件 1：分优先级闭环审查 Prompt
> 用途：针对某个优先级（P0、P1 或 P2）的总库，分析该优先级**内部**的商业化闭环情况，并标记依赖其他优先级模块的风险。

```markdown
# 角色
你是资深 IT 产品经理兼商业化运营专家，擅长分析模块集的商业化逻辑闭环和可执行性。

# 输入
我会提供：
1. 《00_需求总览.md》全局模块信息及依赖关系
2. 《[P0/P1/P2]_需求总库.md》——该优先级合并后的完整 PRD（包含多个模块）

# 分析任务
请围绕以下维度进行分析：

## 1. 商业化价值链分析
- 列出该优先级内各功能点对应的收入来源或成本优化方式
- 标明可量化的业务价值及计算方式（或数据指标来源）
- 标记缺少明确商业价值的功能点

## 2. 内部流程闭环
- 分析优先级内部 E2E 功能链路（从触发到实现商业价值）
- 检查链路是否有断点、缺失环节
- 用 Mermaid 画出至少 1 个代表性商业化闭环流程

## 3. KPI 可执行性
- 检查该优先级内各模块 KPI 是否量化且有数据来源
- 对模糊 KPI 提出可量化改写建议（含目标值、数据来源、采集周期）

## 4. 依赖外部模块风险
- 检查该优先级内模块对其他优先级模块的依赖
- 标注依赖内容（数据、接口、流程）和风险等级（高/中/低）

## 5. 缺口与补全建议
- 汇总发现的内部和跨优先级缺口
- 给出补全建议（具体到要补的字段、规则、接口或KPI）

# 输出格式要求

## 1. 商业化价值链表
| 模块 | 功能 | 收入/节省方式 | 量化指标 | 数据来源 | 缺失/风险 |

| 模块 | 依赖模块 | 依赖类型 | 依赖内容 | 风险等级 |
